3. 全链路日志系统

3.1 使用 CallbackBRWebLogger

final logger = CallbackBRWebLogger(onLog: (entry) => _logs.insert(0, entry.toString()));

BRWebContainerPage(
  logger: logger,
  onUiRequest: (action, _) => logger.uiRequest(action, null),
  onTitleRequest: (title) => logger.native('setTitle', detail: title),
)

3.2 任意位置写日志

logger.native('支付成功', detail: '¥99.00');
logger.native('API', detail: 'POST /login → 200');
logger.uiRequest('hideTabBar', null);

3.3 自动捕获的事件

事件触发方式标签
WebView 生命周期容器内部自动📡 loadStart / loadStop / progress
Bridge 请求/响应BRWebBridge 自动⬆️ REQ / ⬇️ RES
H5 ConsoleonConsoleMessage📜 message
H5 JS 错误onReceivedError💥 error
网络变化BRWebNetworkMonitor🦴 Network changed
自定义logger.native()🦴 任意文本

6. 网络监听 & 系统信息

6.1 网络监听

final monitor = BRWebNetworkMonitor(
  logger: logger,
  onChanged: (status) => logger.native('Network: $status'),
);
monitor.start();

H5 查询：call('device.network.status') → { status: "wifi" }

6.2 系统信息

final info = await BRWebSystemInfo.collect(logger: logger);
DefaultBRWebCapabilityHandler.systemInfo = info;
// → iPhone 15 Pro / iOS 18.0 / app v1.0.0(42)

H5 查询：call('device.system.info') → 完整设备/系统信息

7. 开发期合约检查 (Dev Guard)

BRWebContainerPage(
  capabilityHandler: BRWebDevGuard(
    inner: myHandler,
    logger: _logger,
  ),
  // ← Debug 模式下，H5 调 ui.hideTabBar 但没绑 onUiRequest → 打印 ⚠️ 警告
)
作用：Debug 模式下自动检测回调是否绑好。H5 调了 event 但 Native 没绑回调 → 立即警告，防止带 bug 发布。

10. 扩展业务能力

class AppCapabilityHandler implements BRWebCapabilityHandler {
  @override
  Future<Object?> handle(BuildContext context, BRWebBridgeMessage message) async {
    return switch (message.action) {
      'user.getToken'   => {'token': 'your-token'},
      'payment.create'  => _createPayment(message.params),
      _ => BRWebCapabilityHandlerResult.notHandled,
    };
  }
}

BRWebContainerPage(
  capabilityHandler: CompositeBRWebCapabilityHandler([
    AppCapabilityHandler(),
    DefaultBRWebCapabilityHandler(),
  ]),
);
BRWebDevGuard 包裹： 建议在 handler 外层用 BRWebDevGuard 包裹，Debug 模式下自动检查回调完整性。
