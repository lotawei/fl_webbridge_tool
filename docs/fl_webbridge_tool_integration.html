fl_webbridge_tool 集成手册

把 Flutter App 做成可承载业务网页的壳，并提供 BR_Web 双向通信、生命周期记录、拍照、录音、文件选择和容器关闭能力。

0. 插件仓库结构

fl_webbridge_tool 使用 Flutter 官方插件仓库形态，根目录是插件本体，example/ 是一个独立宿主工程。

fl_webbridge_tool/
  android/        # Android 插件注册代码
  ios/            # iOS 插件注册代码
  lib/            # Dart API 和 BR_Web 容器实现
  example/        # 示例宿主 App，使用 fl_webbridge_tool: path: ../
  docs/           # 集成手册和方案文档
  pubspec.yaml    # 包含 flutter.plugin.platforms

1. 适用场景

Flutter 壳工程 原生 TabBar、路由、登录态、权限和日志由 Flutter 管。
网页业务容器 高频业务页面放在 H5 / Web 页面里，通过 Bridge 调原生能力。
可插拔能力 业务项目可以注册自己的 token、分享、支付、定位等 handler。

2. 添加依赖

如果本地调试当前插件仓库，在外部业务工程中可以使用 path 依赖：

dependencies:
  fl_webbridge_tool:
    path: ../fl_webbridge_tool

当前仓库自带的 example/ 用的是父目录依赖：

dependencies:
  fl_webbridge_tool:
    path: ../

如果发布到 Git 仓库，可以这样接：

dependencies:
  fl_webbridge_tool:
    git:
      url: ssh://your-git/fl_webbridge_tool.git
      ref: v0.1.0

3. Flutter 页面接入

线上 URL 接入：

import 'package:fl_webbridge_tool/fl_webbridge_tool.dart';

BRWebContainerPage(
  title: '业务页面',
  initialUrl: 'https://your-domain.example/page',
  onLifecycle: (event) {
    // 上报 APM、埋点、Debug console
  },
);

离线包或本地 assets 接入：

BRWebContainerPage(
  title: '离线业务页面',
  initialFile: 'assets/h5/demo.html',
);
示例工程已经完成集成，入口在 example/lib/main.dart，依赖配置是 fl_webbridge_tool: path: ../。

4. 网页侧调用 Flutter

网页侧统一调用 BR_WebNativeBridge，请求体包含 id、action、params。

window.BR_WebContainer = {
  call(action, params = {}) {
    const message = {
      id: `${Date.now()}_${Math.random().toString(16).slice(2)}`,
      action,
      params
    };
    return window.flutter_inappwebview.callHandler(
      'BR_WebNativeBridge',
      message
    );
  },
  __nativeCall(payload) {
    console.log('Flutter 调用网页', payload);
    return { received: true };
  }
};

内置 action

await BR_WebContainer.call('device.camera.takePhoto', {
  quality: 80,
  maxWidth: 1600
});

await BR_WebContainer.call('device.file.pick', {
  multiple: true
});

await BR_WebContainer.call('device.audio.startRecord');
await BR_WebContainer.call('device.audio.stopRecord');

await BR_WebContainer.call('container.close', {
  reason: 'h5_request'
});

5. Flutter 主动调用网页

在 onCreated 中拿到 BRWebBridge，之后可以调用网页方法。

BRWebBridge? bridge;

BRWebContainerPage(
  initialUrl: 'https://your-domain.example/page',
  onCreated: (createdBridge, controller) {
    bridge = createdBridge;
  },
);

await bridge?.callWeb('native.ping', {
  'time': DateTime.now().toIso8601String(),
});

6. 扩展业务能力

业务工程只需要实现 BRWebCapabilityHandler，未处理的 action 返回 BRWebCapabilityHandlerResult.notHandled。

class AppCapabilityHandler implements BRWebCapabilityHandler {
  @override
  Future<Object?> handle(
    BuildContext context,
    BRWebBridgeMessage message,
  ) async {
    if (message.action == 'user.getToken') {
      return {'token': 'your-token'};
    }
    return BRWebCapabilityHandlerResult.notHandled;
  }
}

BRWebContainerPage(
  initialUrl: 'https://your-domain.example/page',
  capabilityHandler: CompositeBRWebCapabilityHandler([
    AppCapabilityHandler(),
    DefaultBRWebCapabilityHandler(),
  ]),
);

8. 验证清单

  • 运行 flutter pub get，确认 example 依赖到 fl_webbridge_tool。
  • 运行 flutter analyze，确认静态分析通过。
  • 运行 flutter test，确认 example smoke test 通过。
  • 真机打开 BR_Web tab，观察日志页是否记录 created、loadStart、loadStop、console。
  • 真机分别测试拍照、选择文件、录音开始/停止、关闭容器。

9. 性能建议

  • Tab 场景使用 IndexedStack 缓存 WebView，避免频繁销毁重建。
  • 大文件不要通过 Bridge 传 base64，优先传 path、id 或上传任务 token。
  • 日志要采样上报，高频 console 和 Bridge 调用不要全量打到线上。
  • 权限按需申请，不要在容器初始化时一次性索取。
  • SPA 页面要关注 history 变化，不能只依赖传统页面加载事件。
当前示例基于 flutter_inappwebview 6.x，建议 Android / iOS 都用真机完成能力验收。
