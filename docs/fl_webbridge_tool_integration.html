0.2 最小接入

import 'package:fl_webbridge_tool/fl_webbridge_tool.dart';
import 'package:flutter/material.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  DefaultBRWebCapabilityHandler.systemInfo = await BRWebSystemInfo.collect();
  runApp(MaterialApp(
    home: BRWebContainerPage(
      title: '业务页',
      url: 'https://your-domain.example/page',
      logger: CallbackBRWebLogger(onLog: (e) => print(e)),
    ),
  ));
}
⚠️ 关键：每个 BRWebContainerPage 实例必须传入 capabilityHandler（或通过 BRWebContainerPage 的构造函数传入），否则使用空默认实例，所有 manager 为 null。

2.1 统一消息模型 v0.3

H5 ↔ Native 双向使用 同一种消息格式：{id, action, params, meta?}。响应统一为 {id, ok, data/error}。

方向格式
H5 → Native{id, action, params, meta?} → 响应 {id, ok, data/error}
Native → H5{id, action, params} → H5 可返回 {id, ok, data/error}

2.2 元信息 meta（自动注入）v0.3

H5 每次 brCall 自动携带 meta 字段，包含版本、分支、平台等：

// 默认自动注入
meta: {
  h5Version: "2.0.1",      // __BR_Data__.extra.resourceVersion
  h5Branch: "master",       // __BR_Data__.extra.h5Branch
  platform: "ios",          // navigator.userAgent 自动检测
  appVersion: "1.0.0",     // __BR_Data__.extra.appVersion
  lang: "zh",              // __BR_Data__.lang
}

// 业务扩展（登录后追加 userId/role）
import { setBridgeMeta } from 'br-web-bridge-vue'
setBridgeMeta({ userId: '1001', role: 'admin' })
// 后续所有请求的 meta 自动包含 userId + role
Flutter 端日志中展示 meta：⬆️ REQ [takePhoto] {} meta={h5Version: 2.0.1, platform: ios}

4. 数据库 CRUD v0.2

// 初始化（main 中）
workOrderManager = WorkOrderManager(logger: logger);
await workOrderManager.init();
await workOrderManager.seedIfEmpty();  // 首次自动插入 6 条示例数据

// 注入到 handler

await call('database.workOrder.getById', { id: 1 });           // → {row: {...}}
await call('database.workOrder.insert', {                       // → {id: 7}

6.3 安全 setState v0.2

void _safeSetState() {
  if (!mounted) return;
  if (SchedulerBinding.instance.schedulerPhase == SchedulerPhase.persistentCallbacks) {
    WidgetsBinding.instance.addPostFrameCallback((_) {
      if (mounted) setState(() {});
    });
    return;
  }
  setState(() {});
}

13. 扩展业务能力

class AppCapabilityHandler implements BRWebCapabilityHandler {
  @override
  Future<Object?> handle(BuildContext context, BRWebBridgeMessage msg) async {
    return switch (msg.action) {
      'user.getToken'   => {'token': 'your-token'},
      'payment.create'  => _createPayment(msg.params),
      _ => BRWebCapabilityHandlerResult.notHandled,
    };
  }
}

BRWebContainerPage(
  capabilityHandler: CompositeBRWebCapabilityHandler([
    AppCapabilityHandler(),
    DefaultBRWebCapabilityHandler(),
  ]),
);

14. 常见陷阱

陷阱表现修复
dev server 页面没传 handler DB/网络/资源调用返回 null/unknown BRWebContainerPage(..., capabilityHandler: _handler)
图片选择器死循环 选完图又弹出选图器（重复多次） v0.2 已移除 _pickWithMaxSize 循环逻辑
setState 在 build 阶段崩溃 setState() called during build 使用 _safeSetState() 加 schedulerPhase 检查
DB 错误前端看不到 {ok:true, data:{error:"..."}} v0.2 改为 throw → {ok:false, error:"..."}
音频预览显示 unknown stopRecord 返回缺少 mimeType v0.2 已补 mimeType/name/size 字段
日志 OOM 全局日志无限增长 BRWebGlobalLog.maxCapacity = 2000
