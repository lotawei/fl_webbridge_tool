🔌 Bridge 通信协议

H5 → Native（统一消息模型）

window.flutter_inappwebview.callHandler(
  'BR_WebNativeBridge',
  {
    id: 'req_1712345678_abc',
    action: 'device.camera.takePhoto',
    params: {
      quality: 80,
      maxWidth: 1600,
      saveToGallery: true
    },
    meta: {                    // 自动注入
      h5Version: '2.0.1',
      platform: 'ios',
      appVersion: '1.0.0'
    }
  }
)

Native 响应

// ✅ 成功
{
  "id": "req_1712...",
  "ok": true,
  "data": {
    "path": "/tmp/xxx.jpg",
    "name": "photo.jpg",
    "mimeType": "image/jpeg",
    "size": 204800
  }
}

// ❌ 失败
{
  "id": "req_1712...",
  "ok": false,
  "error": "permission_denied"
}

Native → H5（统一推送，同格式）

// Dart — 与 H5→Native 相同格式
BRWebBridge.callWeb('container.lifecycle', {
  type: 'loadStop', url: '...'
});
// → { id: \"n_xxx\", action: \"container.lifecycle\", params: {...} }

// H5 注册接收 + 可返回响应
onNativeCall((msg) => {
  // msg: { id, action, params, meta? }
  if (msg.action === 'container.lifecycle') {
    handleLifecycle(msg.params);
  }
  // 可返回响应给 Native
  return { id: msg.id, ok: true, data: {} };
});

⛓️ 核心模式：责任链

sequenceDiagram participant H5 participant Bridge as BRWebBridge participant Chain as CompositeBRWebCapabilityHandler participant Biz as 业务 Handler participant Default as DefaultBRWebCapabilityHandler participant Camera as 系统相机 Note over H5: H5 调用自定义能力 H5->>Bridge: callHandler('user.getToken', params) Bridge->>Chain: handle(context, msg) Chain->>Biz: handle(context, msg) Biz-->>Chain: { token: 'xxx' } Note over Chain: ✅ 匹配，直接返回 Chain-->>Bridge: data Bridge-->>H5: { ok: true, data: { token: 'xxx' } } Note over H5: H5 调用设备能力 H5->>Bridge: callHandler('device.camera.takePhoto', params) Bridge->>Chain: handle(context, msg) Chain->>Biz: handle(context, msg) Biz-->>Chain: notHandled Note over Chain: ⏭️ 不匹配，放行 Chain->>Default: handle(context, msg) Default->>Camera: 打开相机 Camera-->>Default: XFile Default-->>Chain: { path, name, mimeType, size } Chain-->>Bridge: data Bridge-->>H5: { ok: true, data: { path: '...' } }

接口定义

abstract interface class BRWebCapabilityHandler {
  Future<Object?> handle(
    BuildContext context,
    BRWebBridgeMessage message,
  );
}

组装使用

BRWebContainerPage(
  capabilityHandler:
    CompositeBRWebCapabilityHandler([
      AppCapabilityHandler(),
      DefaultBRWebCapabilityHandler(),
    ]),
)

自定义能力示例

class AppCapabilityHandler implements BRWebCapabilityHandler {
  Future<Object?> handle(BuildContext context, BRWebBridgeMessage msg) {
    return switch (msg.action) {
      'user.getToken'   => {'token': 'xxx'},
      'payment.pay'     => _pay(msg.params),
      _                 => Future.value(BRWebCapabilityHandlerResult.notHandled),
    };
  }
}

⚡ 关键设计决策与踩坑

决策原因
flutter_inappwebview 而非 webview_flutter 需要 addJavaScriptHandler 支持双向通信
责任链模式 而非 switch 注册 业务可插拔扩展，默认兜底
initialUserScripts AT_DOCUMENT_START 保证同步读取，非 evaluateJavascript fire-and-forget
ImagePicker 一次性选取 + maxWidth 控制 避免循环重试导致选图器反复弹出
错误必须 throw 而非 return error map Bridge 通过 try/catch 区分 ok/fail，return 会被包成 data
媒体四字段统一 path/name/mimeType/size 统一，前端类型推断可用
onLoadStart 补充注入 SPA 路由跳转不触发 AT_DOCUMENT_START

常见陷阱速查

⚠️ handler 必须注入

不传时内部创建空实例，所有 manager 为 null

⚠️ setState 在 build 阶段

需判断 schedulerPhase 并推迟到 postFrameCallback

⚠️ Vue bridge 重入循环

需要 processing 防重入锁 + 按钮 disabled

⚠️ iOS WKWebView dealloc

5 个强引用逐一断开 + loadHTMLString("about:blank")

⚠️ 网络状态首查 'unknown'

需调 checkNow() 实时查询

⚠️ 错误 return vs throw

return error map → 被包成 {ok:true, data:{error:...}}
